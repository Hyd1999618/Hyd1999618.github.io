继承


继承是面向对象三大特征之一（封装、继承、多态）。可以使得子类具有父类的属性和方法，还可以在子类中重新定义，追加属性和方法。

  • Fu：是父类，也被称为基类、超类
  • Zi：是子类，也被称为派生类

    继承中变量的访问特点

  • 在子类方法中访问一个变量
    子类局部范围找$\Rightarrow$子类成员范围找$\Rightarrow$父类成员范围找$\Rightarrow$如果都没有就报错（不考虑父亲的父亲……）
    （成员方法的访问特点也是这个顺序）

  • super关键字
    与this关键字用法相似
    this：代表本类对象的引用
    super：代表父类存储空间的标识（可以理解为父类对象引用）

方法重写

  • 私有方法不能被重写（父类私有成员子类是不能继承的）
  • 子类方法访问权限不能更低（public > default > private）

【注意】：用private修饰的类、属性、方法，只能自己使用，别的类是不能访问的

权限修饰符的访问情况

final关键字

final关键字是最终的意思，可以修饰成员方法、成员变量、类

final修饰的的特点

  • 修饰方法：表面该方法是最终方法，不能被重写
  • 修饰变量：表面该变量是常量，不能再次被赋值
  • 修饰类：表面该类是最终类，不能被继承


final修饰局部变量

  • 变量是基本类型：final修饰指的是基本类型的数据值不能发生改变
  • 变量是引用类型：final修饰指的是引用类型的地址值不能发生改变，但是地址里面的内容是可以发生改变的

基础知识

jvm：java虚拟机，针对不同的操作系统提供不同的jvm（java能够跨平台的核心）。有了 Java 虚拟机的帮助，我们编写的 Java 源代码不必再根据不同平台编译成对应的机器码了，只需要生成一份字节码，然后再将字节码文件交由运行在不同平台上的 Java 虚拟机读取后执行就可以了
jre：JRE是Java Runtime Environment的缩写，顾名思义是java运行时环境，包含了java虚拟机，java基础类库。是使用java语言编写的程序运行所需要的软件环境，是提供给想运行java程序的用户使用的，还有所有的Java类库的class文件，都在lib目录下，并且都打包成了jar。
jdk：JDK是Java Development Kit的缩写，顾名思义是java开发工具包，是程序员使用java语言编写java程序所需的开发工具包，是提供给程序员使用的。JDK包含了JRE，同时还包含了编译java源码的编译器javac
javac XXX：编译环境
java XXX：运行环境


jvm $\subset$ jre $\subset$ jdk

switch-case语句

case穿透：除非case语句块中有break;，程序会跳出switch选择结构，否则继续执行下一个case，一直这样执行下去，直到整个switch语句全部执行完毕。
产生这种现象的原因:由于某个case语句中缺少或者漏写break语句所导致的结果.

搭建hexo+GitHub博客遇到的问题及解决

发表于 2021-10-18 更新于 2021-10-20

这是这几天在谷歌、百度、CSDN、知乎、B站……的帮助下，用hexo+GitHub搭建了一个静态个人博客，然后在阿里云买了一个域名（主页：www.heyundong.xyz）。因为一直觉得自己有好多东西要学，但是只看网上那些文字或者视频总觉得运用不起来。比如之前看过git、markdown这种，但是看了就忘而且基本也用不到，这次搞这个博客也是逼自己去学习和练习这些有意思的东西，并可能会记录自己的一些杂事吧 。

我的邮箱：1084350802@qq.com


搭建这个博客过程中也是跟着网上的教程一步一步来的，但是肯定不止我一个人碰到过，就是跟着教程走着走着，步骤就对不上了，或者别人显示的结果和自己的结果不一样。

  • 一些教程直接原封不动搬运别人。

    连显示的结果图也是抄别人的，当然不会去验证其教程的正确性。可能原作者一开始在某个地方写错了，但是别人还是直接搬运，很离谱。


  • 版本、系统甚至电脑型号的不同。

    版本不同太常见了，我看别人的教程时，总是很害怕和别人的版本不一样（自己看到教程时的那个最新版本一般比作者使用的版本要高），虽然基本上都是向下兼容，但是有时候还是很害怕。尤其是我一开始只会对照图片一步一步点击的时候。这种情况我觉得小白的话还是最好用和作者一样的版本，先用起来再说，那些最新版的什么新功能一开始也用不到。

    系统不同就是windows、mac、linux啥的，这个除非自己搜索错了吧，我是用windows基本搜索出来常见的就是Windows下面的教程，其他的比如mac搜索时加上关键词就好了。Linux我也没怎么用过，用这个的感觉都是大佬，应该可以找到方法解决吧。

    版本型号不同我遇到的很少，只有一次安装ubuntu时，网上说啥开机按什么什么进入BIOS设置，但是不同的电脑进入那个设置之后的界面是不一样的。这种情况也可以加搜索的关键词解决，还是解决不了的话，只能硬着头皮试了，这种由于型号不同，虽然过程可能不太一样，但也没多大差别。


  • 可以找教程视频

    对，指的就是我这种人。搭建这个博客的时候遇到了很多问题，幸好我找到了一个几个月之内发布的教程视频，作者也指出了GitHub的一些改动（默认main分支和令牌形式的登录），并且也说网上一些教程没有更新。我觉得遇到这种情况也算是少数，最主要还是视频的话至少保证作者是一步一步做的，不像第一条当中那些人直接搬运看都不看。我一般也就是在B站上面找，YouTube里面也有，但被英文无字幕劝退了。


搭建博客的具体过程就不搬运了，我自己也是一步一步跟着做的，也不是很明白其中原理，说一些我遇到的问题和解决方法吧，当然这些解决方法基本搜索也能搜出来。

问题及解决

1.下载nodejs后会让我们在node安装目录创建node_global和node_cache文件夹。我当时在写npm命令时，写错了文件路径，然后后来再用npm这个命令的时候就开始报错，我把nodejs卸载了重新装都不行

手动更改“C:\Users\Administrator.npmrc”文件(这个Administrator每个人是不一样的，”Users“可能显示的是中文”用户“)，在记事本中打开，将文本内容修改为上教程里面说的安装的正确的配置路径。

2. 这个报错记不清了，一大堆单词，但是出现了 error 4048 ，这个我的问题是git装在C盘，得用管理员身份运行，不然有的功能用不了。在开始菜单搜索栏搜索”git bash“，然后右击”以管理员身份运行“，但这个时候进入的是C盘目录，这时候用cd命令进到自己的Blog目录即可。

3. 后续操作中，在某个文件夹执行

4. 由于目前GitHub默认分支变成main分支，所以将本地的master分支也改成main，这个我后面遇到了一些错误，不知道是不是由于没改名的缘故，但改了肯定更稳

6. 对博客进行操作时，hexo s后，通过localhost访问会立刻更新，但是hexo g通过GitHub.io或者自己的域名访问时，会有一定的延迟，不要着急。

7.买域名之后，直接解析，根据教程，一个@一个www，记录值写XXX.github.io（网上有的说要写ping的地址，我没用到，直接这样写就行），然后就是到GitHub上面的page填自己的域名，然后save，这时GitHub项目下面会生成一个CNAME文件。这时一定要注意！！！在本地的Blog/source文件夹下面，新建一个无后缀名的CNAME文件，其中的内容和GitHub那个CNAME里面的内容一样。如果不新建这个文件，每次更新博客，都得重新save域名，很麻烦。
